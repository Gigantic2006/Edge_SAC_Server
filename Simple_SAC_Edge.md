# 今天要实现：极简版边缘调度SAC

## 1. 环境
- 节点数：2
- 用户数：1
- 任务：每时隙随机到达，数据量[10,40] Mbits，计算密度200 Gcycles/Mbit
- 节点算力：节点0算力10 GHz，节点1算力20 GHz
- 状态：状态：[任务大小，节点0积压，节点1积压]
- 动作：选节点（只有一个）
- 奖励：-（计算时延 + 排队时延）

## 2. SAC
- Actor: 输入3维，输出2维（动作概率）
- Critic: 输入5维（状态+动作），输出1维
- 隐藏层：[128, 128]

## 3. 训练
- episodes: 500
- 每episode: 50个时隙
- 每步存储经验
- 每64步更新一次网络

## 4. 验证
- 打印每个episode的总成本
- 看是否收敛

## 5. 具体的公式
- 状态[任务大小，节点0积压，节点1积压]
- 动作 2维one-hot
- 奖励 -(计算时延+排队时延)
- 计算时延公式: 
   $\frac{数据量*计算密度}{节点算力}$
- 排队时延公式: 

   $T_{wait}=\frac{当前节点的积压}{节点算力}$
- 队列积压更新公式:

   $q_j(t) = \max\left\{ q_j(t-1) + \mathbb{1}\{l_n^s(t) = j\} C_n(t) - f_j \Delta, \ 0 \right\}$
